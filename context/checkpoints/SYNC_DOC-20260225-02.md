# Resumen de Cambios Técnicos - Fase 2 Módulos

## 1. Cambios en Base de Datos (`schema.prisma`)

Se requiere agregar dos nuevos modelos y un enum para soportar la nueva funcionalidad.

**a) Nuevo Modelo: `Appointment`**
Gestiona la logística previa a la atención médica.
- Relación con: `Branch`, `Worker`, `ServiceProfile`.
- Relación 1:1 opcional con `MedicalEvent` (una cita completada genera un evento).
- Campos clave: `scheduledStart`, `scheduledEnd`, `status`.

**b) Nuevo Enum: `AppointmentStatus`**
Valores: `SCHEDULED`, `CONFIRMED`, `CANCELLED`, `NO_SHOW`, `COMPLETED`.

**c) Nuevo Modelo: `AuditLog`**
Registro inmutable de seguridad.
- Relación con: `User`.
- Campos JSON `details` para guardar diffs (valor anterior vs nuevo).

## 2. Instrucciones para SOFIA

1.  **Actualizar Prisma:**
    - Copiar los modelos del SPEC `ARCH-20260225-06-FASE2-MODULOS.md` al archivo `frontend/prisma/schema.prisma`.
    - Ejecutar `npx prisma migrate dev --name add_appointments_and_audit` para aplicar cambios.

2.  **Lógica de Negocio (Backend/Actions):**
    - Crear `src/services/appointment-service.ts`: Lógica de creación y validación de horarios.
    - Crear `src/services/audit-service.ts`: Función `logAction(userId, action, entity, entityId, details)` para ser llamada desde otros servicios.
    - Crear `src/actions/dashboard-actions.ts`: Consultas `prisma.groupBy` y `prisma.count` para los KPIs.

3.  **UI Checklist (Frontend):**
    - **Dashboard:** Crear `/app/dashboard/page.tsx` con tarjetas de resumen.
    - **Citas:** Crear `/app/citas/page.tsx` (Lista/Calendario) y `/app/citas/new/page.tsx`.
    - **Check-in:** Botón en la cita que invoque la Server Action `convertAppointmentToEvent`.

4.  **Integración de Auditoría:**
    - En `MedicalEventService.updateStatus`, insertar llamada a `auditService.logAction`.
    - En las Server Actions de Login (NextAuth), intentar registrar el acceso si es posible (via hook o callback).
